## 四层交叉分析体系

```
原始新闻
   ↓
┌─────────────────────────────┐
│  🐵 悟空（深度分析）          │  ← S级 + A级 优质新闻
│  输入：S/A级新闻             │
│  输出：市场情绪 + 操作策略     │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│  🧅 沙僧（散户情绪）          │  ← 全量新闻（含噪音）
│  输入：7天全部新闻 × 时间衰减  │
│  输出：散户追高/恐慌概率       │
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│  🐴 白龙马（主力行为）        │  ← K线 + 量价 + 悟空/八戒信号
│  输入：沙僧情绪 + K线 + 悟空/八戒│
│  输出：吸筹/出货/洗盘 + 矛盾检测│
└─────────────────────────────┘
   ↓
┌─────────────────────────────┐
│  🐷 八戒融合（最终决策）      │  ← 悟空 + 沙僧 + 白龙马
│  输入：悟空分析+沙僧情绪+白龙马│
│  输出：融合后胜率+行动+ETF组合 │
└─────────────────────────────┘
```

### 三层交叉分析（核心设计）

1. **白龙马融合悟空/八戒信号**：矛盾检测 + 可信度调整
   - 悟空乐观 + 白龙马出货 → 矛盾⚠️ 可信度×0.7
   - 悟空恐慌 + 白龙马吸筹 → 逆向机会📌 可信度×1.2
   - 八戒胜率高 + 白龙马出货 → 胜率降级
   - 八戒胜率低 + 白龙马吸筹 → 逆向布局

2. **八戒融合沙僧/白龙马信号**：胜率+行动调整
   - 散户狂热 + 八戒胜率>60% → 追高即被套，胜率降25%
   - 散户恐慌 → 逆向机会，胜率+15%
   - 主力出货 + 散户追高 → 坚决减仓
   - 主力吸筹 → 分批布局（优先级最高）

3. **信号一致性评分**：3分=共振 / 2分=正常 / 1分=混乱

## 新闻分类标准

| 级别 | 定义 | 影响时长 | 标记 |
|------|------|---------|------|
| S级 | 影响整个市场的重大新闻 | 1-4周 | 🔴 |
| A级 | 影响特定板块的重要新闻 | 1-3天 | 🟡 |
| 抖音 | 社交媒体热点、平台动态 | 短期 | 📱 |

### 操作类型
- **可做** → ✅ 机会明显
- **警惕** → ⚠️ 有风险需谨慎
- **回避** → ❌ 建议回避
- **关注** → 📌 观察等待

## 自动化流程

### 数据流
```
1. 搜索新闻（10-20条全部新闻）
2. 保存 all_news（全量）+ 筛选 S/A级（优质）
3. 悟空分析 S/A级 → wukong_judgment
4. 八戒分析 S/A级 → bajie_conclusion（原始）
5. 运行 daily_update_with_models.py：
   - 沙僧处理 all_news → sangsha_module
   - 白龙马处理 K线+沙僧+悟空+八戒 → white_dragon
   - 八戒融合：沙僧+白龙马 → bajie_conclusion（最终）
6. 生成 news-data.js
7. git push 到 wnews 仓库
8. 微信/飞书/网站三路推送
```

## 🔧 遇到的问题与解决方案（2026-04-11）

### 问题1：S/A级新闻要点未显示
- **现象**：S/A级新闻卡片只显示 summary，没有显示 key_point
- **根因**：`renderNewsCard` 函数渲染 `item.summary`，而五维进化系统生成的是 `item.key_point`
- **解决方案**：
  ```javascript
  // 修复前
  text = item.summary || '无摘要';

  // 修复后
  text = item.key_point || item.summary || '无摘要';
  ```
- **Commit**：8efed08

### 问题2：唐僧仲裁模块未渲染
- **现象**：五维进化系统生成了 `tang_sanzang` 数据，但前端页面没有显示
- **根因**：`index.html` 完全没有唐僧模块的渲染代码
- **解决方案**：
  1. 新增 `.tang-seng-section` CSS样式（金色主题）
  2. 在白龙马模块之后添加唐僧渲染代码
  3. 显示：仓位/最终行动/仲裁矛盾/仓位公式/风控触发
- **Commit**：8efed08

### 问题3：review.html 语法错误导致加载卡死
- **现象**：复盘页面一直显示"正在加载复盘数据..."
- **根因**：第421行 `sort()` 比较函数少一个闭合括号
  ```javascript
  // 错误
  const wdEnt=Object.entries(wdStates).sort((a,b)=>b[1]-a[1);
  // 正确
  const wdEnt=Object.entries(wdStates).sort((a,b)=>b[1]-a[1]);
  ```
- **解决方案**：修复语法错误
- **Commit**：abf96b2

### 问题4：逐日准确率表格排版问题
- **现象**：表格行高过高，内容竖排堆叠，没有实际走势列
- **根因**：样式设计不合理，缺少实际走势列
- **解决方案**：
  1. 删除"关键事件"列（S级事件数）
  2. 新增"实际走势"列（绿色字体）
  3. 改为紧凑横排布局
  4. 添加金色表头 + hover金色高亮
- **Commit**：2c6aae1

### 问题5：4.11新闻来源混淆
- **现象**：wnews 和 news-website 两个网站的4.11新闻一模一样
- **根因**：wnews 没有独立自动化任务，混用了 news-server 的数据源
- **解决方案**：
  1. 创建 wnews 独立自动化任务（wnews-9-2, wnews-22, wnews-weekly）
  2. 使用 `news-evolution/` 目录（v4系统）
  3. 完全隔离两套系统：
     - news-website → news-server/（v3系统）
     - wnews → news-evolution/（v4五维进化系统）

### 问题6：唐僧模块样式不突出
- **现象**：唐僧模块使用紫色主题，与其他模块区分不明显
- **根因**：初始设计时未考虑模块重要性层级
- **解决方案**：
  1. 改为金色主题（唐僧 = 最高决策权）
  2. 加粗金色边框（2px）
  3. 添加金色光晕效果
- **Commit**：922ecb1

### 🎯 经验1：地缘事件驱动行情真实存在，但窗口期极短（1-3天）

**验证案例**：3月29日霍尔木兹危机 → 油气/黄金暴涨，完美捕捉。

**核心逻辑**：霍尔木兹海峡封锁导致全球20%石油运输中断，缺口无法短期弥补，价格必然飙升。

**窗口期经验**：
- 地缘驱动行情通常在消息确认后1-3天内达到峰值
- 之后要么持续升级，要么快速反转（意外缓和时）
- 4月5日的失误恰恰是没有为「反转」留后手

**可复刻公式**：
```
地缘事件 + 供需缺口不可弥补 = 高胜率做多窗口（快进快出）
```
**关键变量**：事件是否会导致持续性供给缺口？

## 🚨 失败教训：4月5日判断错误根因分析

> **每次失败都是系统升级的机会。以下教训必须永久记忆。**

### 失误全过程

1. **4月5日预判**：伊朗拒绝Trump48小时通牒 → 局势将持续升级 → 油气/黄金继续强势
2. **4月6日实际**：美伊在48小时截止前达成停火协议 → 避险逻辑秒反转
3. **后果**：若在4月5日追高黄金/油气，在4月6日将面临3-5%浮亏

### 根因三要素

| 根因 | 说明 | 改进措施 |
|------|------|---------|
| **线性外推谬误** | 伊朗强硬拒绝 → 判断局势升级。但地缘政治多方博弈，结果非线性 | 嘴炮升级≠实际升级；真正的升级信号是军事行动而非嘴炮 |
| **信息更新速度缺失** | A股清明休市无法实时更新先验概率 | 地缘先验每6-12小时更新一次 |
| **止损线类型单一** | 只设价格止损-7%，没设逻辑止损 | 地缘事件的硬条件止损：停火协议/和平声明/海峡开放 |

### 改进后的操作框架（三项规则）

#### 规则①：地缘判断修正
```
✅ 正确信号：双方开始实际军事行动 / 经济制裁落地
❌ 错误信号：双方互相强硬表态 / 外交威胁升级
💡 规律：「双方互相强硬」往往意味着谈判桌快到了
📌 窗口期：地缘驱动行情1-3天内必须快进快出
⛔ 硬止损：停火协议达成 / 关键人物和平声明 → 立即平仓
```

#### 规则②：节假日前风控
```
若持仓逻辑依赖地缘/大宗商品：
├─ 清明/五一/国庆等长假前
├─ 评估：「休市期间有哪些重要信息可能落地？」
├─ 若有高概率落地的事件 → 大幅减仓或清仓
└─ 原因：休市期间信息不对称风险极大
```

#### 规则③：贝叶斯动态更新
```
├─ 地缘政治先验概率极不稳定
├─ 关键变量设为价格触发条件而非持仓条件
└─ 每6-12小时重新评估先验概率
```

## 🧠 修正后的悟空分析框架（成功范式）

> **这是经过17天实盘验证后提炼的核心决策框架，所有后续分析必须遵循此框架。**

### 四大铁律

```python
# ① 地缘事件铁律
if event_type == "geopolitical":
    if signal == "verbal_escalation":     # 嘴炮升级
        action = "DO_NOT_FOLLOW"           # 不跟
        window = "SHORT_1_3_DAYS"          # 窗口期极短
    elif signal == "military_action":      # 实际行动
        action = "TRADE_WITH_CAUTION"      # 谨慎参与
        logic_stop = True                  # 必须设逻辑止损

# ② 节假日前铁律
if is_holiday_approaching():
    if holding_logic_depends_on("geopolitics") or holding_logic_depends_on("commodities"):
        action = "REDUCE_POSITION"         # 降低敞口
        reason = "信息不对称风险"

# ③ 贝叶斯更新铁律
if prior_type == "GEOPOLITICAL":
    update_frequency = "6_12_HOURS"        # 高频更新
    key_variable_as = "TRIGGER"            # 关键变量=触发条件
    # NOT: key_variable_as = "HOLD_REASON" # 绝不是持仓理由

# ④ 铁律指标体系
leading_indicators = {
    "US_NASDAQ_FUTURES": "lead_A_stock_by_1_day",   # 纳指期货领涨A股1天
    "BTC_PRICE": "lead_crypto_concept_by_1_day",    # BTC领涨加密概念1天
    "NIKKEI_INDEX": "lead_A_stock_by_1_day",        # 日经领涨1天
}
# 重点监控：节假日前一天的外盘走势
```

### 成功范式提纯

| 场景 | 成功模式 | 失败模式 |
|------|---------|---------|
| 地缘事件 | 快进快出 + 逻辑止损 | 持仓等待 + 仅价格止损 |
| 财报季 | 业绩龙头 + 行业周期拐点 | 跟风炒作 + 无基本面支撑 |
| Trump喊话 | 逆向思维（喊多→警惕） | 跟随喊话方向 |
| 先行指标 | 节假日前看外盘做预判 | 忽略外盘信号 |
| 贝叶斯 | 动态更新先验概率 | 固定先验不调整 |

### 信号一致性决策树

```
四层信号全部一致(3分) → ✅ 坚定执行
三层一致(2分) → ✅ 执行但控制仓位
两层一致(1分) → ⚠️ 减半仓位或不做
全部矛盾(0分) → ❌ 完全回避，等待明确信号
```
