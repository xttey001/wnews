# 🧠 复盘进化系统（wnews-review）v2

## 一、核心理念

> **「每个徒弟各精其职 + 不犯同样的错误 + 让成功变得高效」= 五维进化体系**

复盘的本质是**知识回流到正确的人身上**：
- ❌ 传统做法：所有经验堆给唐僧，徒弟们原地踏步
- ❌ 错误做法：只在八戒融合层做调整，悟空/沙僧/白龙马不变
- ✅ 进化做法：**谁负责的领域 → 谁的经验 → 谁来进化**

1. **4月5日地缘失误**：线性外推「伊朗拒绝→局势升级」，忽略了博弈论中的「极限施压=谈判前奏」

### 3.2 🧔 沙僧进化 — 韭菜直觉更精准

#### 当前职责
- 统计7天累积的散户情绪（买入概率/恐慌概率/持有概率）
- 时间衰减机制（越近的日期权重越高）
- 输出整体情绪评级（狂热/积极/平稳/恐慌/极度恐慌）

#### 复盘发现的提升空间
1. **狂热信号滞后**：当沙僧已经标记「狂热」时往往已经晚了，需要更早预警
2. **恐慌中的逆向信号不够突出**：散户恐慌往往是最好的买点，但这个信号没有被充分强调
3. **情绪拐点预判缺失**：只能描述当前状态，不能预判即将转折

### 3.5 🙏 唐僧 — 只做协调（精简版）

#### 唐僧的职责（重新定义）

唐僧**不是包揽一切的总管**，而是**裁判员**：

| 做 | 不做 |
|----|-----|
| 跨层矛盾仲裁（徒弟们打架时裁决） | 替徒弟做他们专业领域内的判断 |
| 全局风控检查（节假日/黑天鹅） | 重复计算徒弟们已经算过的东西 |
| 最终决策格式化和输出 | 积累自己的独立分析能力 |
| 记录决策结果供后续验证 | 存储经验（经验存在各徒弟的知识库里）|

#### 唐僧的仲裁规则

    all_aligned: "四层共振，坚定执行",

## 四、经验回流机制（闭环设计）

### 4.1 每次分析的完整生命周期

```
Day T（分析日）
  │
  ├─ 1. 新闻输入
  │
  ├─ 2. 各模块带经验执行
  │   ├─ 🐵悟空 ← 注入 WX-001~003 分析经验
  │   ├─ 🧔沙僧 ← 注入 SS-001~003 情绪经验  
  │   ├─ 🐉白龙马 ← 注入 WD-001~003 主力经验
  │   └─ 🐷八戒 ← 注入 BJ-001~003 概率经验
  │
  ├─ 3. 🙏唐僧协调（如有矛盾）
  │
  ├─ 4. 输出最终决策
  │
  └─ 5. 存入待验证队列（decision_id + 预判内容 + 时间戳）
  

Day T+N（验证日，N=1~5天）
  │
  ├─ 6. 获取实际走势数据
  │
  ├─ 7. 对比预判 vs 实际
  │   ├─ ✅ 判断对了 → 提纯成功范式 → 更新对应模块的经验库
  │   └─ ❌ 判断错了 → 记录失败教训 → 更新对应模块的铁律
  │
  └─ 8. 生成复盘报告（review.html 数据来源）
```

### 4.2 经验归属规则

**核心原则：谁犯的错 → 谁记教训；谁的成功 → 谁得经验**

| 判断错误类型 | 归属模块 | 示例 |
|-------------|---------|------|
| 方向判断错了（涨看成跌） | 🐵 悟空 | 地缘事件误判为升级→实际缓和 |
| 胜率估算偏差大 | 🐷 八戒 | 预估75%胜率→实际只有40% |
| 情绪信号遗漏/延迟 | 🧔 沙僧 | 没提前预警狂热→等标记时已晚 |
| 主力状态误判 | 🐉 白龙马 | 把洗盘看出货→过早离场 |
| 跨层矛盾处理不当 | 🙏 唐僧 | 徒弟打架时听错人了 |

### 遇到的问题与解决方案

#### 问题1：S/A级新闻要点未显示
- **现象**：S/A级新闻卡片只显示 summary
- **根因**：渲染字段错误（应显示 `key_point`）
- **解决方案**：`text = item.key_point || item.summary || '无摘要'`
- **Commit**：8efed08

#### 问题2：唐僧仲裁模块未渲染
- **现象**：生成了 `tang_sanzang` 数据但前端不显示
- **根因**：缺少渲染代码
- **解决方案**：新增唐僧模块渲染代码 + 金色主题CSS
- **Commit**：8efed08, 922ecb1

#### 问题3：review.html 加载卡死
- **现象**：一直显示"正在加载复盘数据..."
- **根因**：JS语法错误（少闭合括号）
- **解决方案**：修复 `sort()` 比较函数语法
- **Commit**：abf96b2

#### 问题4：逐日准确率表格排版问题
- **现象**：行高过高，缺少实际走势列
- **解决方案**：删除关键事件列 + 新增实际走势列 + 紧凑横排布局
- **Commit**：2c6aae1

#### 问题5：4.11新闻来源混淆
- **现象**：wnews 和 news-website 新闻数据相同
- **根因**：wnews 没有独立自动化任务
- **解决方案**：创建独立自动化任务 + 使用 news-evolution/ 目录
- **结果**：两套系统完全隔离
  - news-website → news-server/（v3系统）
  - wnews → news-evolution/（v4五维进化系统）

#### 问题6：唐僧模块样式不突出
- **现象**：紫色主题区分不明显
- **解决方案**：改为金色主题 + 加粗边框 + 光晕效果
- **Commit**：922ecb1
