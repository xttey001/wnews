## 早报问题诊断与修复

### 问题诊断结果

#### 问题1：为什么把4.13的新闻放到4.14号
**结论**：这是**预期行为**，不是bug。

**原因**：
- 早报在早上9点运行时，新闻源通常还没有当天的新内容
- `search_today.py` 会创建当天的新闻文件（如 `news_2026-04-14.json`）
- 但内容是昨天的新闻，因为早上9点市场刚开盘

**数据结构**：
- `news_2026-04-13.json`：包含完整的4.13早报+晚报数据
- `news_2026-04-14.json`：包含4.12-4.13的新闻（用于4.14早报）

---

#### 问题2：为什么唐僧没有给出唐憎结论
**结论**：字段缺失bug，已修复。

**原因**：
- `daily_update_v4.py` 生成的 `tang_sanzang` 字段缺少 `唐憎结论` 字段
- 前端 `index.html` 第1245行期望这个字段：`const tsNote = ts.唐憎结论 || ts.conclusion || '';`

**修复内容**：
```python
# daily_update_v4.py 第375-390行
risk_summary = ''
risk_checks = tang_result.get('risk_checks', [])
high_risks = [c for c in risk_checks if '高风险' in c.get('status', '') or '必须' in c.get('status', '')]
if high_risks:
    risk_summary = f'（注意：{len(high_risks)}项高风险）'

tang_conclusion = f'{tang_action_text}，胜率{tang_wr_text}{risk_summary}'

data['tang_sanzang'] = {
    '仓位': tang_position,
    '最终行动': tang_action_text,
    '唐憎结论': tang_conclusion,  # 【新增】
    ...
}
```

**验证结果**：
- `news_2026-04-14.json` 现在包含：`"唐憎结论": "双主线配置：石油ETF+芯片ETF，胜率~50%（注意：1项高风险）"`
- 网站已更新：https://xttey001.github.io/wnews/
